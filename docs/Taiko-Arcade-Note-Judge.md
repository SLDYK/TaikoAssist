# 太鼓达人街机 — 音符类型与判定机制

> 本文档基于太鼓达人（太鼓の達人）官方街机版本整理，同时参考了本项目中 `TaikoChartData.NoteType` 枚举与 `GlobalSettings` 判定参数的设计。

---

## 一、输入设备：太鼓

街机使用实体太鼓，鼓面分为两个感应区域：

| 区域 | 日文 | 输入判定 | 对应颜色 |
|------|------|----------|----------|
| 鼓面（中心） | ドン (Don) | 敲击鼓面 | 🔴 红色 |
| 鼓边（边缘） | カッ (Kat) | 敲击鼓边 | 🔵 蓝色 |

一台街机通常支持 **双人游玩**（1P / 2P），每人各使用一面鼓。

本项目中，键盘映射为四个区域：

| 区域 | 默认按键 |
|------|----------|
| 左 Don | `F` |
| 右 Don | `J` |
| 左 Kat | `D` |
| 右 Kat | `K` |

## 二、音符类型一览

### 2.1 标准音符（瞬时型）

敲击一次即完成判定。

| 名称 | 日文 | TJA 字符 | NoteType 枚举值 | 图示 | 操作 | 基础得分 |
|------|------|----------|----------------|------|------|----------|
| 咚（小） | ドン | `1` | `Don = 1` | 🔴 小红圈 | 敲鼓面 | ~300 |
| 咔（小） | カッ | `2` | `Kat = 2` | 🔵 小蓝圈 | 敲鼓边 | ~300 |
| 大咚 | 大ドン | `3` | `BigDon = 3` | 🔴 大红圈 | 敲鼓面 | ~600 |
| 大咔 | 大カッ | `4` | `BigKat = 4` | 🔵 大蓝圈 | 敲鼓边 | ~600 |

> 大音符的判定窗口比普通音符更宽松（见 [判定窗口](#三判定窗口)）。

### 2.2 连打 / 长条型音符（持续时间型）

需要在一段时间内连续敲击。

| 名称 | 日文 | TJA 字符 | NoteType 枚举值 | 说明 |
|------|------|----------|----------------|------|
| 连打（黄条） | 連打 (Renda) | `5` 起 `8` 止 | `Roll = 5` | 黄条期间不断敲鼓面，尽量多打。无最低次数要求，打得越多分越高。 |
| 大连打 | 大連打 | `6` 起 `8` 止 | `BigRoll = 6` | 连打的加大版，判定窗口更宽。 |
| 风船 | 風船 (Balloon) | `7` 起 `8` 止 | `Balloon = 7` | 气球音符，必须在时间内敲够指定次数才能"打破"。次数由 `BALLOON` 指令指定。 |
| 九素玉 | くすだま (Kusudama) | `9` 起 `8` 止 | `Kusudama = 9` | 与风船类似，但外形为球形。也需达到指定击打次数。 |

**连打得分规则**：每次击中连打获得约 300 分（大连打约 600 分），无次数上限。

**风船/九素玉得分规则**：打破后获得固定奖励分（通常 5000 分），若未打破则 0 分。

### 2.3 特殊音符（分支谱 / 里谱面）

这些音符通常出现在分支谱或隐藏谱面（裏譜面）中：

| 名称 | 日文 | TJA 字符 | NoteType 枚举值 | 说明 |
|------|------|----------|----------------|------|
| 手拍子·红 | Hand Don | `A` | `HandDon = 10` | 大咚 + 拍手属性，双人配合谱使用 |
| 手拍子·蓝 | Hand Kat | `B` | `HandKat = 11` | 大咔 + 拍手属性 |
| 地雷 | 地雷 (Mine) | `C` | `Mine = 12` | 不可敲击！击中会爆炸扣分 |
| 导火线 | 導火線 (Fuse) | `D` | `Fuse = 13` | 类似风船连打的特殊连打 |
| 即兴音符 | アドリブ (Adlib) | `F` | `Adlib = 14` | 装饰性音符，不计分，不判 MISS |
| 花咚 | 花どん (Kadon) | `G` | `Kadon = 15` | 特殊特效音符 |

## 三、判定窗口

太鼓达人街机的判定分为 **良 / 可 / 不可** 三级：

```
         ←── 良 ──→
    ←──────── 可 ────────→
                          ←──── 不可（MISS）────
```

### 3.1 判定等级

| 判定 | 日文 | 含义 | 得分倍率 | 连击 |
|------|------|------|----------|------|
| **良** | 良 (Ryo) | 精准命中 | 100% | 计入 COMBO |
| **可** | 可 (Ka) | 命中但偏差较大 | 50%（部分版本） | 计入 COMBO |
| **不可** | 不可 (Fuka) | 未命中或偏差过大 | 0 | COMBO 中断 |

### 3.2 判定窗口时间（默认值）

本项目 `GlobalSettings` 中的默认值（参考街机标准）：

| 参数 | 默认值 | 适用范围 |
|------|--------|----------|
| `JudgeGood` | **±25ms** | 所有音符的"良"判定 |
| `JudgeOkNormal` | **±75ms** | Don / Kat / Roll / Balloon 等普通音符的"可"判定 |
| `JudgeOkBig` | **±108ms** | BigDon / BigKat / BigRoll 等大音符的"可"判定 |

示意图：

```
普通音符： [  良 ±25ms  ][     可 ±75ms      ][        不可        ]
大音符：   [  良 ±25ms  ][       可 ±108ms        ][      不可      ]
```

> **注意**：不同街机版本（如 新筐体 / 旧筐体 / 虹版 / にじいろ）的精确判定帧数可能有微小差异，以上为通用参考值。

## 四、连打/风船的特殊判定

### 4.1 连打（Roll / BigRoll）

- 连打开始后，在连打持续时间内每次敲击鼓面都算作一次有效连打。
- 无最低次数要求；时间结束后自动结束。
- 每次有效敲击独立计分，同时也受 **良/可** 判定窗口约束。
- 连打中敲鼓边（Kat）无效。

### 4.2 风船（Balloon / Kusudama）

- 必须在指定时间内敲够 `RequiredHits` 次鼓面。
- 达到次数 → **打破成功**，获得奖励分（如 5000 分）。
- 未达到次数 → **打破失败**，0 分。
- 每次敲击同样需要落在判定窗口内（连打速度越快越好）。

## 五、连打/风船的"头判"机制

### 5.1 官方街机：无头判

在太鼓达人官方街机中，**连打（Roll）和风船（Balloon）类音符没有"头判"机制**。

- 连打/风船的音符从 `StartTime` 开始即进入"可敲击"状态，第一次敲击可以发生在开始后的任意时刻（只要在结束前）。
- 不存在"必须在开始时刻的判定窗口内击中头部"的要求。
- 击中头部与击中身体的判定逻辑完全相同，仅根据当时的时间偏移判定 **良/可**。

### 5.2 社区模拟器的头判变体

部分 TJA 模拟器（如 TJAPlayer3 的某些分支）引入了可选的**头判（Head Judgment）**机制：

- 连打/风船音符的第一次敲击被单独区分，必须在音符起始时刻附近（判定窗口内）命中才算"头部通过"。
- 若头部 MISS，整个连打/风船可能被判定为失败或大幅扣分。

> ⚠️ 这是社区模拟器的**可选规则**，并非官方街机行为。是否启用通常由设置项控制。

### 5.3 本项目中的头判设计

本项目当前不启用头判机制。连打/风船类音符在整个 `[StartTime, EndTime]` 区间内均可自由敲击，无头部特判逻辑。

## 六、判定流程（本项目实现）

本项目的判定流程位于 `InputDetect`、`NoteCreator` 和 `RendaCreator` 中：

1. **时间推进**：`Timer` 类跟踪谱面播放进度（`elapsed` 秒）。
2. **音符生成**：`NoteCreator` 和 `RendaCreator` 在音符接近判定线时从对象池取出并显示。
3. **按键检测**：`InputDetect` 监听四个鼓面区域的键盘输入。
4. **判定匹配**：
   - 标准音符：检查 `|elapsed - NoteTime| < JudgeWindow`。
   - 连打/风船：检查是否在 `[StartTime, EndTime]` 范围内且满足次数要求。
5. **结果**：根据偏差时间落在哪个窗口内，给出 **良 / 可 / 不可** 判定。
